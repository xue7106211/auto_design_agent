# 适配结果验证

对多终端适配生成的设计稿进行规则符合性检查，输出通过/不通过判定和具体偏差项。不修改设计稿，只做检查和报告。

## 适用场景

- 主 Skill 生成完成后的自动验证
- "帮我检查这个折叠屏适配稿是否正确"
- "验证一下布局尺寸"
- "校验组件属性是否符合规范"

## 前置条件

执行前需确认以下信息（由主 Skill 传入或由用户提供）：

- 目标 frame 的节点 ID
- 目标设备类型（Fold / Pad）
- 布局类型（NLC / NC / LC / C）
- 预期的尺寸参数（画布尺寸、栏宽、边距）

## 强制工作流

### Step 1：采集目标状态

1. 用 `get_metadata` 获取目标 frame 的完整图层结构
2. 用 `get_screenshot` 获取目标 frame 的视觉截图
3. 加载对应的规则文档：
   - `references/common-rules.md`（通用规则）
   - `references/layout-lc-nc.md` 或 `references/layout-nlc.md` 或 `references/layout-c.md`（对应布局规则）

### Step 2：结构校验

逐项检查以下内容，记录每项的实际值和预期值：

**画布级**：
- [ ] 目标 frame 宽度 = 预期设备宽度
- [ ] 目标 frame 高度 = 预期设备高度

**状态栏**：
- [ ] 全局状态栏数量 = 1
- [ ] 状态栏宽度 = frame 全宽
- [ ] 状态栏位于 frame 顶部

**布局结构**（按布局类型检查）：

LC 布局：
- [ ] 左栏宽度 = 预期值
- [ ] 右栏宽度 = 预期值
- [ ] 左栏 + 右栏 + 分割线 = 内容区总宽

NLC 布局：
- [ ] 导航栏宽度 = 预期值
- [ ] 列表栏宽度 = 预期值
- [ ] 内容区宽度 = 预期值
- [ ] 导航项数量 = 源页面底部 Tab 数量

C 布局：
- [ ] 内容区宽度 = 预期值
- [ ] 左右边距 = 预期值

**节点层级**：
- [ ] 顶层节点数量合理（状态栏 + 主内容区，不超过 3-4 个）
- [ ] 无孤立节点（不在任何容器内的游离元素）

### Step 3：视觉校验

基于截图检查：

- [ ] 无元素重叠
- [ ] 无裁切错误（内容被意外截断）
- [ ] 无异常留白（大面积空白区域）
- [ ] 无异常拉伸（图片或组件比例失调）
- [ ] 栏间分割清晰（有分割线或间距）
- [ ] 整体视觉平衡（左右内容量大致均衡）

### Step 4：语义校验

- [ ] NLC / NC 布局：悬浮底部导航栏已移除，转为侧边导航
- [ ] LC 布局：悬浮底部导航栏按规则处理（保留或 Q18 特殊处理，参考 `references/layout-lc-nc.md`）
- [ ] C 布局：悬浮底部导航栏保留在内容区底部悬浮展示
- [ ] 移动端底部手势条已移除
- [ ] 移动端键盘弹层已移除
- [ ] 不存在重复的状态栏
- [ ] NLC 布局：导航选中项与列表内容语义一致
- [ ] NC 布局：导航选中项与内容区语义一致
- [ ] LC 布局：列表选中项与详情内容语义一致

### Step 5：生成报告

汇总所有检查项，生成验证报告。

## 输出格式

```
## 验证结果：通过 / 不通过

### 设备：{Fold / Pad}  布局：{NLC / NC / LC / C}

### 通过项
- ✓ 画布尺寸：888 × 2009dp（符合预期）
- ✓ 左栏宽度：356dp（符合预期）
- ...

### 不通过项
- ✗ 右栏宽度：实际 520dp，预期 532dp（偏差 -12dp）
- ✗ 存在异常留白：右栏底部约 80dp 空白区域
- ...

### 修复建议
1. 将右栏宽度从 520dp 调整为 532dp
2. 检查右栏底部操作区是否缺失，补充或调整内容填充
```

## 判定规则

- 所有结构校验项通过 + 无严重视觉问题 = 通过
- 尺寸偏差 ≤ 2dp 视为通过（浮点精度容差）
- 任何一项结构校验不通过 = 不通过
- 存在重叠或裁切 = 不通过
- 轻微留白（< 20dp）可标记为警告，不判定为不通过

## 注意事项

- 本 Skill 只做检查和报告，不修改设计稿
- 修复操作由调用方（主 Skill 或用户）根据修复建议执行
- 如果缺少预期参数（如用户未指定设备类型），先从 `references/device-dimensions.md` 推断，推断不了则要求用户补充
