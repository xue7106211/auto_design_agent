# NLC 布局规则（Navigation-List-Content 三栏）

本文档定义 NLC（导航-列表-内容）三栏布局在各设备上的适配规则。

> 以下为骨架结构，具体数值和规则需根据小米设计规范填写。

## 适用场景

NLC 布局仅适用于 **Pad**（横屏 / 竖屏），不适用于 Fold。适配对象为手机端有悬浮底部导航栏（Bottom Tab Bar）的应用——转为 NLC 三栏后，手机端悬浮底部导航栏转为侧边导航栏（N 栏），原悬浮底部导航栏移除。

典型应用：笔记、文件管理、录音机等具备"多 Tab 切换 + 列表 + 详情"结构的应用。

## 栏宽与 padding

栏宽和 padding 的具体数值见 `device-dimensions.md`，本文档仅定义适配逻辑。

## 导航栏（N 栏）适配规则

### 悬浮底部导航栏 → 侧边导航转换

手机端悬浮底部导航栏在 NLC 布局中转为侧边导航栏（N 栏），原悬浮底部导航栏移除：

- 导航项数量和顺序与手机端底部 Tab 保持一致
- 当前选中态与列表栏内容对应
- 悬浮底部导航栏的详细尺寸和排列规则见 `device-dimensions.md` 悬浮底部导航栏章节

### 侧边导航包含内容

侧边导航栏包含三类内容：

1. **一级导航**：同手机底部 Tab
2. **某一级导航的二级功能**
3. **用户在一级导航中创建的内容**

其中 2 和 3 可来自不同一级导航。

### 侧边/顶部导航切换

- 顶部导航仅支持展示侧边导航的一级功能
- 侧边选择非一级导航后切换为顶部导航，选中其所属一级导航

### 展开/收起

- **默认状态**：Pad 横屏 / 竖屏均默认展开态
- **展开态**：显示图标 + 文字标签（N 栏宽度见 `device-dimensions.md`）
- **收起态**：仅显示图标（N 栏宽度见 `device-dimensions.md`）
- 收起态默认仅支持展示侧边导航的一级功能
- 侧边选择非一级导航后切换为收起态，收起态会保留选中的非一级导航同时隐藏其他非一级导航
- 如果收起的侧边栏有二级功能，则此二级功能一直显示，直到应用进程结束，或被其他二级功能替代

### 悬浮侧边栏

- 所有 N 栏在显示悬浮侧边栏时，N 栏内部左侧产生 6dp padding（组件自身携带）

## 列表栏（L 栏）适配规则

### 顶部模块

列表栏顶部放置来自手机端首页的顶部模块：

- 标题栏（Pad-TopBar）
- 搜索栏
- 分段控件（SegmentedControls）
- 选项卡（SelectableChip）

### 列表内容

- 列表填充顶部模块以下的剩余视窗
- 移除手机端悬浮底部导航栏、底部提示器等不需要的移动端元素（底部 Tab 已转为 N 栏）
- 列表栏语义为纯侧边栏，不保留移动端底部导航语义

### 列表选中态

- 列表选中项与内容区详情内容语义一致

## 内容区（C 栏）适配规则

### 顶部

- 放置详情导航栏（Pad-TopBar）

### 正文

- 建立正文视窗 frame
- 放入正文内容（clone 手机端详情页内容区）
- 通过视窗裁剪表达当前可视区域，不保留完整长页

### 底部

- 固定底部操作区（如 AI 输入框）
- 删除移动端状态栏、底部手势条、键盘弹层等不需要的元素

## 栏间分割线

- 分割线为右侧栏 inside stroke 1px
- 具体见 `device-dimensions.md` 栏间分割线定义

## 横竖屏旋转规则

NLC 框架横竖屏旋转规则见 `device-dimensions.md` 横竖屏旋转规则章节。

核心要点：
1. 展开态对展开态，收起态对收起态
2. 横竖屏各自独立记录侧边栏状态
3. 不能旋转出模态导航

## 特殊处理

- 导航栏选中项、列表栏内容、内容区详情三者语义必须一致
- 空状态：列表为空时，内容区显示空状态占位
