# Figma Foldable Layout Adaptation

本文档由 `skill-main-workflow.md` 在需要读取历史折叠屏适配规则时按需读取。
本文档不是独立 Skill，不直接触发执行；它只提供折叠屏或大屏分栏页面适配的参考规则。

在 **现有目标 frame 内** 完成折叠屏或大屏分栏页面适配，不新建并行页面，不先讲方案，直接执行。

## 适用场景

当主流程遇到以下类型需求且需要历史折叠屏规则补充时读取本文档：

- “把两个移动端页面适配成折叠屏页面”
- “把首页和详情页重组成左右分栏”
- “在现有目标 frame 里完成大屏/横屏布局”
- “复用现有状态栏变体、列表变体完成设备适配”
- “不要重画，直接基于现有设计稿重组”

## 核心原则

1. 先探查，后修改。
2. 优先复用已有组件、变体、画布节点。
3. 能 clone 已落地节点时，不优先 create。
4. 所有写入必须分步执行，不要一次性大脚本。
5. 每一步写入后必须做截图校验和结构校验。
6. 如果实例化受字体或组件限制，立即切换为 clone 策略。
7. 只保留目标设备当前视口语义，不保留移动端长滚动页语义。
8. 只做局部修正，不整页推翻重做。

## 强制工作流

### Step 1: 读取上下文

先获取这些信息：

- 目标 frame metadata
- 源页面 metadata
- 相关组件集 metadata
- 目标设备对应变体 metadata
- 必要的 screenshot
- 必要的 design context

必须先确认：

- 哪些节点直接复用
- 哪些节点替换为目标变体
- 哪些节点删除不迁移

### Step 2: 选择实现策略

按以下优先级执行：

1. 直接复用目标变体
2. clone 画布上的现成节点
3. 最后才使用 Plugin API 新建节点

如果出现以下情况，立即切换到 clone 策略：

- `createInstance()` 失败
- `appendChild()` 因字体问题失败
- 组件依赖不可用字体
- 实例内部文本难以稳定修改

切换规则：

- 放弃组件实例化
- clone 已在画布中的现成节点
- 优先改布局、尺寸、位置，不改组件内部结构

### Step 3: 分步写入

不要一次性完成全部内容，至少拆成以下阶段：

#### Phase A: 搭目标骨架

- 清空目标 frame 子节点
- 设置目标 frame 尺寸
- 如果目标设备为 Fold 内屏 Q18，设置目标 frame 四角圆角为 `50dp`
- 建立顶部全局状态栏
- 建立主内容区
- 建立左右内容区 frame
- 为视口容器设置 `clipsContent`

#### Phase B: 填充左栏

- 放入首页顶部模块
- 替换或放入目标列表变体
- 删除不需要的移动端底部导航/底部指示器
- 让左栏成为纯侧边栏语义
- 让列表填充顶部模块以下的剩余视口

#### Phase C: 填充右栏

- 放入详情导航
- 建立正文视口 frame
- 放入正文内容
- 固定底部 AI 输入区
- 删除移动端状态栏、底部杆子、键盘弹层

### Step 4: 每一步后做校验

#### A. 截图校验

每次关键写入后调用 screenshot，检查：

- 是否有重叠
- 是否有裁切错误
- 是否有异常留白
- 左右栏宽度是否视觉正确
- 顶部状态栏是否只有一套
- 底部 AI 输入区是否固定到底部

#### B. 结构校验

每次关键写入后读取结构，检查：

- 目标 frame 尺寸
- Fold 内屏 Q18 目标 frame 圆角是否为 `50dp`
- 状态栏尺寸
- 左右栏尺寸
- 顶层节点数量
- 顶层节点位置
- 关键节点是否位于正确层级

### Step 5: 修正原则

如果结果不符合预期，按以下顺序修：

1. 先修尺寸
2. 再修位置
3. 最后修文本或局部视觉

不要整页重做，只做局部修正。

## 输出要求

最终只输出这些内容：

- 已完成的结构变化
- 实际尺寸结果
- 是否满足验收项
- 是否存在妥协项

不要输出长篇方案说明，不要输出泛泛设计建议。

## 默认验收标准

- 目标页面尺寸精确匹配需求
- 左右栏宽度精确匹配需求
- 顶部全局状态栏只有一套
- 左栏不保留移动端底部导航语义
- 右栏保留正文与 AI 输入区，但不保留键盘弹层
- 左栏选中项与右栏内容语义一致
- 整体视觉无明显裁切、重叠、异常空白

## 推荐实现偏好

对于这类任务，优先采用以下映射方式：

- 顶部区域：直接复用目标设备状态栏变体
- 左栏：源首页顶部模块 + 目标列表变体
- 右栏：源详情导航 + 源详情正文 + 底部 AI 区
- 正文区域：通过视口裁切表达当前可见区域，而不是保留完整长页

## 注意事项

- 如果目标 frame 已存在，默认直接覆盖其内部内容
- 不新建“V2”“副本”“新页面”等并行设计稿，除非用户明确要求
- 尽量减少对组件内部文字和结构的修改
- 当目标只是设备适配时，优先重排，不优先重构视觉
