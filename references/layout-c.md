# C 布局规则（Content 通栏）

本文档定义 C（通栏）布局在各设备上的适配规则。

> **负责人**：老唐
> **Kim 修改建议**：基于 Figma 应用框架页、适配规则页、`device-dimensions.md` 实测补充。

## 适用场景

C 布局适用于 **Fold 内屏**（横屏 / 竖屏）和 **Pad**（横屏 / 竖屏）。适配对象为手机端不需要分栏的单一内容页。

典型应用：设置页、关于页、单一详情页、播放器全屏页等。

## 栏宽与 padding

具体数值见 `device-dimensions.md`，以下为速查摘要。

### Fold 内屏

| 方向 | 内容区宽度 | 左右 padding |
|------|----------|-------------|
| 横屏 | 全宽（888dp） | 56dp |
| 竖屏 | 全宽（628dp） | 12dp |

### Pad

| 方向 | 内容区最大宽度 | C 栏 padding |
|------|-------------|-------------|
| 横屏 | 1100dp（超出后居中，两侧边距 Fill） | 56+161dp（超过 1100 按 1100 的 padding + 剩余宽度） |
| 竖屏 | 全宽（949dp） | 56dp |

## 适配规则

### 宽度调整

- 内容区宽度拉伸至目标设备宽度，受左右 padding 和最大宽度约束
- Pad 横屏内容超过 1100dp 时居中展示，两侧边距 Fill
- 不要自作主张将通栏改为分栏

### 边距重算

- 按 `device-dimensions.md` 中对应设备的 C 栏 padding 值重算左右边距
- 内部模块的左右边距同步调整

### 内容模块

- 按源页面从上到下的顺序，逐模块迁移
- 各模块宽度适应新的内容区宽度
- 删除不需要的移动端元素（底部手势条、键盘弹层等）

## 悬浮底部导航栏

C 通栏布局中，悬浮底部导航栏保留在内容区底部悬浮展示，不转为侧边导航。

- 悬浮底部导航栏的尺寸、tab 宽度、对齐规则、FAB 关系等详见 `device-dimensions.md` 悬浮底部导航栏章节
- 与 NLC / NC 布局不同，C 通栏不存在"底部 Tab → 侧边导航"转换

## 特殊处理

- 空状态：内容为空时显示空状态占位
- 通栏页面保持通栏，不要改为分栏或网格
