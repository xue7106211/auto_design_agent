## 断点定义

应用 全屏/分屏/小窗 共用同一套规则，根据「窗口宽度」和「设备类型」进行判断，布局自动适应。

| 断点 | 容器宽度 | 内容区左右 padding | 代表设备 |
|------|---------|-----------------|--------|
| w < 600 | 0–599dp | 8dp | Phone 竖屏、Pad 1/3 分屏、Fold 外屏、Flip 外屏 |
| 600 ≤ w < 960 | 600–959dp | 16dp | Pad 竖屏、Pad 1/2 分屏、Pad 2/3 分屏 |
| 600 ≤ w < 960（Fold 专属） | 600–959dp | 16dp | Fold 内屏 |
| w ≥ 960 | 960+ dp | 44dp | Pad 横屏、Pad 2/3 分屏 |

> 所有 N 栏在显示悬浮侧边栏时，N 栏内部左侧会产生 6dp 的 padding（由组件自身携带）。

## 横竖屏旋转规则（Pad / Fold 通用）

NLC 与 NC 框架均适用以下规则：

1. **展开态映射**：横屏展开态 ↔ 竖屏展开态，收起态 ↔ 收起态
2. **独立历史记录**：横竖屏各自记录用户干预后的侧边栏状态
3. **状态循环**：横屏收起 →❶ 竖屏展开 →❷ 竖屏收起 →❸ 横屏展开 →❹ 横屏收起
4. **禁止旋转出模态导航**：旋转不能触发导航模态切换

## 悬浮底部导航栏（Bottom Tab Bar）

底部导航栏为**悬浮样式**，不占据页面布局空间，悬浮覆盖在内容区底部之上。

### 尺寸

| 设备 / 方向 | 高度 | 说明 |
|-----------|------|------|
| 手机竖屏 | 100dp | — |
| 手机横屏（横版 pill） | 88dp | 左右结构，总长度 = 各 tab 长度累加，单个 tab 最大宽度 166dp |
| Fold 内屏（横屏 / 竖屏） | 100dp | — |
| Fold 外屏竖屏 | 100dp | N8 外屏最多 2 个 tab |
| Fold 外屏横屏 | 侧置，宽度 100dp | — |
| Pad（横屏 / 竖屏） | 100dp | — |

### tab 宽度规则

| 场景 | 宽度策略 |
|------|---------|
| 手机竖屏 / Fold 外屏竖屏 | 铺满屏幕宽度，左右 padding 各 24dp |
| Fold 内屏 / 横屏 / Pad | 屏幕宽度 ≥ 440dp → 固定总宽 **344dp**；< 440dp → 铺满，左右 padding 各 24dp |

### tab 数量与排列

| tab 数量 | 排列方式 |
|---------|---------|
| 2 ~ 3 个 | 每个 tab 固定 88dp，居中排列 |
| 4 ~ 5 个 | 居左紧凑排列 |

### 对齐规则

| 方向 | 有 fab | 对齐方式 |
|------|--------|---------|
| 竖屏 | — | 居中 |
| 横屏 | 无 | 居中 |
| 横屏 | 有 | 沿右侧对齐 |

### 特殊情况

1. **Q18 LC 栏 + 底导航并存时**：底导航与 fab 分开展示，fab 置于最右侧，底导航控制在左侧 L 栏宽度以内
2. **N8 外屏**：最多只展示 2 个 tab
3. **手机横屏**：为左右结构，总长度 = 各 tab 长度累加；单个 tab 最大宽度不超过 166dp
